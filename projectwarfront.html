Project Overview

Project Warfront is an original 100 player first-person shooter real-time strategy hybrid multiplayer Half-Life: 2 experience on the Source Engine, combining strategic gameplay mechanics with innovative systems. The game emphasizes teamwork, large-scale progression, and competitive depth through unique features that set it apart in the FPS genre.

This project showcases advanced game development capabilities including complex system architecture, real-time networking, and polished user experience design. The only gamemode running the original source engine to accomplish combat at this scale with proper optimization techniques.

Construction System

A diverse advanced class-driven construction system with over 200 placable buildings from defense, logistics, unit deployment, and specialized headquarter structures. This system alone took over two years to create.
  • Faction-specific construction menus expose four build categories tied directly to each class permission tier.
  • Cards enumerate building name, description, cost, power demand, and the zones you are allowed to deploy within.
  • HQs and FOBs operate as the primary build-radius anchors, ensuring every outpost grows outward with intention. The idea was to make FOBs the conquest objective point that could be placed anywhere making each round unique and different.

Building Placement

  • Authorized classes browse the menu, pick a structure, and instantly preview its footprint on terrain.
  • Rotation snap defaults to 15¬∞‚Äîleft/right mouse button nudges allow quick alignment.
  • Press R to begin construction; animated assets play if available, otherwise the structure materializes immediately.
  • Insufficient funds, invalid terrain, or out-of-radius attempts flag the exact reason and block the placement.
  • The ghost mesh glows green when valid and red when the placement rules are violated.

Client-Side Build Settings

  • Players can tweak QoL options in the client menu to match their placement style.
  • Rotation snap can be increased up to 45¬∞ for faster, grid-friendly fortifications.
  • Toggle structure icons to declutter the HUD when multiple squads are building nearby.
  • Disable snap entirely for buttery-smooth placement when organic shapes are preferred.

Critical Structure Cooldowns

  • High-impact builds such as the FOB have a cooldown timer so players with build permissions can't spam power positions giving the enemy team a chance to attack the HQ without instant invulnerability.
  • The UI locks deploy buttons until the cooldown clears, preventing overlapping bubbles and keeping pacing fair.

Placement Validation

  • Ghost projections snap red the moment you leave HQ/FOB range, pairing the visual cue with a text notification.
  • Only when the ghost returns to green‚Äîinside the valid radius and on navigable ground‚Äîdoes the system allow a deploy.
  • Clear messaging reinforces why a placement fails, helping new builders learn the rules without breaking immersion.

Map Progression System

Each map is a territory that your faction must control to progress the war effort. The first faction to capture the opposing faction's main territory wins the war.

  • The routes image is an example representation of what the code is doing. Each map has options for winning and losing directions which links to another map in the list.
  • Destroying an HQ immediately ends the round; the defeated faction forfeits that territory while the attackers choose their next objective.
  • Route voting keeps the war effort to player-driven‚Äîlanes where the collective of the team can chose the best strategic direction as some routes have terrain that favors the losing side but are faster to reach the enemy main territory.

QoL & Misc Systems

Warfront layers countless comfort features that reduce friction, improve readability, and optimize performance. You can find more QoL updates in the Community Discord.

Map Preset System

Curated world-state saves mean developers can reboot a 100-player scenario in minutes‚Äîno more rebuilding structures or vehicle pools after an unexpected restart.

  • Emergency building and entity caching lets staff reload the last autosave state whenever a server crash interrupts a test.
  • Admins can capture curated presets that respawn critical structures, FOB layouts, and vehicle pools on match start to keep iteration fast.

Revive System

Automated Class Selection

Squad leadership and class permissions remain synchronized thanks to the respawn-driven selection flow and automatic squad promotions.

Class Menu Cycle

  • Every respawn surfaces the four primary classes so players can instantly lock the role they need before redeploying.
  • Creating a squad promotes the founder to NCO, unlocking the full construction kit and vehicle deployment capabilities while other members stay on their chosen primaries.
  • Leaving the squad as the leader forces a fresh class pick, and the next player in the list inherits NCO duties automatically.
